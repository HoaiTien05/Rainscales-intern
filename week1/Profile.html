Semantic HTML — Viết code có "tâm hồn"

Ngày đầu học HTML, mình cứ dùng <div> cho tất cả mọi thứ. Mãi về sau mới hiểu rằng <header>, <nav>, <main>, <article>... không chỉ là tên đẹp — chúng mang ý nghĩa thật sự cho cả trình duyệt lẫn người đọc code.

"Một trang web tốt không cần nhìn thấy để hiểu — ngay cả khi đọc code thuần, cấu trúc đã nói lên tất cả."

Đây là bài học mình muốn chia sẻ với bất kỳ ai mới bắt đầu: hãy yêu semantic HTML từ sớm. Nó sẽ giúp bạn viết code sạch, dễ bảo trì và thân thiện với người dùng khuyết tật.
